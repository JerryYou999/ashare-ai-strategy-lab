# PRD v1.0｜A股 AI 策略实验室

## 1. 项目背景
用户需要在极短时间内完成一个具备 AI 能力的产品 demo。结合当前交易算法课程背景，产品选择聚焦于“策略生成 + 回测验证 + 市场情报 + 研究学习”的闭环，而非真实交易执行平台。

## 2. 产品目标
打造一个面向 A 股交易学习者和轻度投资者的 AI 策略实验平台，使用户能够：
1. 从模板或自然语言快速生成交易策略；
2. 在沪深300范围内做日频历史回测；
3. 结合国内外热点新闻理解行业影响；
4. 上传论文/研报，让 AI 提炼出新的候选策略。

## 3. 目标用户
### 3.1 核心用户
- 有基础投资兴趣但不会写量化代码的学生 / 初级交易者
- 想验证交易想法的产品经理、研究助理、课程项目成员

### 3.2 用户痛点
- 不会把自然语言策略转成可执行规则
- 不会快速回测
- 不知道策略是否过拟合
- 读了论文却很难落地为可实验策略
- 看到市场新闻，但不知道和交易策略有什么关系

## 4. 产品定位
一个面向 A 股交易学习者的 AI 策略实验平台，支持模板选策、自然语言生成、历史回测、市场情报分析，以及从论文中自动提炼新策略。

## 5. 成功指标（MVP）
- 用户可在 3 分钟内完成一次策略创建与回测
- 至少 7 个模板可直接运行，1 个 Beta 模板可展示研究方向
- 论文上传后，AI 可输出结构化策略摘要
- 演示路径完整：模板/输入 → 回测 → 解释 → 新闻 → 研究

## 6. MVP 范围
### 6.1 必做
1. 沪深300股票选择
2. 日频回测
3. A 股近似真实规则：只做多、T+1、100 股、佣金、印花税
4. 策略模板市场
5. 自然语言策略生成（依赖 LLM）
6. 回测指标与收益曲线
7. 市场情报摘要与行业影响解释
8. 论文上传与策略提炼

### 6.2 不做
- 真实下单
- 用户账户系统
- 社区跟单
- 分钟级高频回测
- 复杂多空组合实盘
- 完整新闻数据库/搜索引擎

## 7. 产品信息架构
1. Tab1：策略实验室
2. Tab2：模板市场
3. Tab3：回测结果
4. Tab4：市场情报
5. Tab5：策略研究员

## 8. 核心功能设计

### 8.1 Tab1 策略实验室
**目标**：完成策略输入、编辑与参数确认。

**输入项**：
- 股票代码
- 起止日期
- 初始资金
- 风险偏好
- 策略来源（模板 / 自然语言 / 研究员导入）
- 策略类型
- 策略参数

**输出项**：
- 当前策略卡片
- 参数 JSON
- 风险提示

**关键交互**：
- 保存当前策略到 session
- 使用 AI 根据自然语言生成策略配置

### 8.2 Tab2 模板市场
**目标**：降低上手门槛。

**模板清单**：
1. 双均线趋势
2. 布林带均值回归
3. Alexander Filter 突破
4. 短期动量
5. 长期反转
6. 跳空策略
7. 季节效应
8. 新闻事件漂移（Beta）

**每张模板卡包含**：
- 名称
- 适用行情
- 核心逻辑
- 默认参数
- 是否支持自动回测

### 8.3 Tab3 回测结果
**目标**：展示策略效果并帮助判断是否值得继续研究。

**输出指标**：
- 累计收益率
- 年化收益率
- 最大回撤
- Sharpe Ratio
- 交易次数
- 胜率
- 平均持有天数
- Buy & Hold 对照

**图表**：
- 策略净值曲线 vs B&H
- 回撤曲线
- 买卖点散点图（可选）

### 8.4 Tab4 市场情报
**目标**：把“新闻”转成“可行动解释”。

**输入**：
- 国内热点文本
- 海外热点文本
- 当前策略摘要

**输出**：
- 今日市场摘要
- 受影响行业 Top 3
- 对当前策略的影响：利多 / 利空 / 中性
- AI 给出的风险与机会说明

### 8.5 Tab5 策略研究员
**目标**：通过上传论文/研报，提炼新策略。

**输入**：
- PDF / txt / md 文件
- 输出深度（快速摘要 / 策略提炼 / 深度研究）

**输出**：
- 论文一句话摘要
- 结构化策略要素
- 候选策略卡片（原始论文版 / A股简化版 / 当前系统可实现版）
- 一键导入策略实验室

## 9. 用户流程
### 主流程
1. 用户在模板市场选择模板
2. 进入策略实验室调整参数
3. 在回测结果页运行回测
4. 查看市场情报对策略的解释
5. 在研究员页上传论文，生成新策略并再次回测

### 替代流程
1. 用户在策略实验室直接输入自然语言
2. AI 生成策略参数
3. 回测并查看解释

## 10. 业务规则
- 股票范围：沪深300（首版可允许用户手动输入股票代码）
- 回测频率：日频
- 市场规则：只做多、T+1、100 股一手、佣金、卖出印花税
- 行情使用 OHLCV 日线数据
- 新闻事件模板首版可为 Beta，不强制自动回测
- AI 生成的新策略如果超出系统能力范围，则转写为“最接近的可实验版本”

## 11. 风险与约束
1. 新闻模块容易失控，必须控制在“摘要 + 行业映射 + 策略解释”范围内
2. 论文策略常涉及高频、做空、另类数据，必须有“不可自动回测”的兜底逻辑
3. A 股真实规则复杂，MVP 只做近似模拟
4. 不能把 AI 输出包装成投资建议，应明确为“研究和学习用途”

## 15. 验收标准
- 五个 Tab 均可打开
- 至少一个模板可以完整跑通：选股 → 回测 → 指标 → 图表
- 论文上传能输出结构化摘要
- 新闻文本能输出摘要和行业影响
- 可用 Streamlit 成功部署
